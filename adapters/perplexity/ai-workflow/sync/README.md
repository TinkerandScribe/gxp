# Staying Aligned with Core (Perplexity Adapter)

This directory contains documentation for keeping the Perplexity adapter in sync with the canonical GXP methodology in `core/`.

Perplexity is primarily used for **research support** within the GXP workflow (especially Phase 0 / upfront research). Because its role is more specialized than general coding agents (Grok, Claude, Cursor), the sync process is intentionally lighter.

## Current Recommendation

1. Before conducting research that will feed into a task brief or plan, review the latest `../../../core/workflow.md` (focus on Phase 0 and turning research into strong briefs).

2. Compare against the Perplexity-optimized guidance in this adapter (focus-modes.md, prompt-templates.md, research-handoff.md, research-to-brief.md, research-workflow.md, collections-strategy.md).

3. Note any meaningful changes in research philosophy or handoff practices.

4. Decide whether to adopt into the Perplexity versions or consciously diverge (document why in a brief note or this README).

## Future Tooling

This adapter may eventually include a lightweight review checklist or script. For now, alignment is maintained through deliberate scoping, periodic manual review of core/workflow.md, and the research-specific instructions here.

See `../../../core/workflow.md` for the source methodology and `../../README.md` for the overall adapters model. The `collections-strategy.md` here explains how to use Perplexity Collections to compound value over time within GXP.
